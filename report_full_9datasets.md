## 1. 研究背景与目标

### 1.1 问题定义

人体活动识别（Human Activity Recognition, HAR）是普适计算领域的核心任务之一。传统基于传感器数据的HAR方法依赖手工特征或浅层模型，在复杂场景（多类、过渡态、稀缺数据）下泛化能力有限。

### 1.2 研究目标

利用大语言模型（MiniMax-M2.7）作为教师模型，基于传感器窗口的物理特征生成软标签（Soft Labels），通过知识蒸馏（Knowledge Distillation）提升CNN学生模型的分类准确率。

### 1.3 核心假设

> MiniMax大模型具备丰富的物理世界常识（如上下楼的Y轴加速度特征差异、走路vs慢跑的频率差异），这些隐含知识可以作为"教师信号"帮助CNN更好地学习。

## 2. 方法论

### 2.1 知识蒸馏框架

```
传感器窗口 → 物理特征提取 → MiniMax API → 软标签概率分布
                                         ↓
        CNN学生模型 ←── 联合损失函数 ←── 硬标签
           ↓
      预测输出
```

### 2.2 蒸馏损失函数

$$L = \alpha \cdot L_{Focal} + (1-\alpha) \cdot L_{KD}$$

其中：
- $L_{Focal}$: Focal Loss，处理类别不平衡
- $L_{KD} = T^2 \cdot KL(\sigma(z_i/T) \| \sigma(q_i/T))$: KL散度蒸馏损失
- $\alpha = 0.6$（默认），$T = 3.0$（温度参数）

## 4. 训练策略与超参数

### 4.1 优化器与学习率调度

| 参数 | Pure CNN | +MiniMax蒸馏 |
|------|---------|-------------|
| 优化器 | AdamW | AdamW |
| 学习率 | 5e-4 | 5e-4 |
| Weight Decay | 1e-4 | 1e-4 |
| 调度器 | CosineAnnealingWarmRestarts | CosineAnnealingWarmRestarts |
| T₀ | 20 | 20 |
| T_mult | 2 | 2 |

### 4.2 损失函数

**Focal Loss（Pure CNN）**:
$$L_{Focal} = -\sum_i (1-p_t)^\gamma \cdot \log(p_t)$$

其中 $\gamma = 2.0$，用于处理类别不平衡。

**Combined Loss（蒸馏）**:
$$L_{Combined} = 0.6 \cdot L_{Focal} + 0.4 \cdot L_{KL}$$

其中 $L_{KL}$ 使用温度 $T=3.0$ 进行缩放。

### 4.3 数据增强策略

| 阶段 | 增强方式 |
|------|---------|
| Epoch < 40（蒸馏） | 无额外噪声 |
| Epoch ≥ 40（蒸馏） | Gaussian Noise σ=0.02 |
| Epoch < 80（Pure CNN） | Mixup α=0.4 |
| Epoch ≥ 80（Pure CNN） | Gaussian Noise σ=0.02 |

### 4.4 训练规格

| 参数 | 新数据集(step1_pure_cnn/step2_distill) | 全部数据集(run_all_datasets) |
|------|----------------------------------------|------------------------------|
| Epochs | 60 | 100 |
| Batch Size | 64 | 64 |
| 最大训练样本 | 20,000 | 20,000 |
| 验证集比例 | 20%（从train中划分） | 20%（从train中划分） |
| 测试集比例 | 20% | 20% |
| 梯度裁剪 | 5.0 | 5.0 |

### 4.5 类别不平衡处理

- **Focal Loss**: 自动降低易分类样本的权重，聚焦难分类样本
- **每类软标签采样**: 新数据集每类采样50个代表性样本调用MiniMax API
- **数据划分**: 使用`stratify=y`保证训练/验证/测试集类别分布一致

## 5. MiniMax API与软标签生成

### 5.1 API配置

| 参数 | 值 |
|------|-----|
| API Endpoint | `https://api.minimaxi.com/v1` |
| 模型 | `MiniMax-M2.7-highspeed` |
| Timeout | 60秒 |
| Max Tokens | 120 |
| Reasoning Split | True（启用思维链） |

### 5.2 Prompt工程设计

从每个传感器窗口提取以下物理特征：

```
acc_mag = sqrt(acc_x² + acc_y² + acc_z²)
y_mean  = mean(acc_y)
peaks   = number of local maxima in acc_mag
dom_freq = dominant frequency from FFT(acc_mag)
```

**Prompt模板**:
```
Classify IMU window. Classes: {0=下楼, 1=坐着, 2=站立, ...}
Features: acc_mag={:.2f}±{:.2f}, y_mean={:.4f}, peaks={}, freq={:.1f}Hz
Physics: upstairs=posY~1Hz, downstairs=negY~1Hz, walk=posY~1-2Hz, jog=high~2-4Hz, sit/stand=low~0Hz
Output JSON: {{"0":p0, "1":p1, ...}}
```

### 5.3 软标签解析策略

1. **首选**: 从JSON格式输出中解析概率值
2. **备选**: 从MiniMax推理过程中提取概率值
3. **兜底**: 若API失败或解析失败，使用one-hot硬标签

### 5.4 温度T的作用

- $T=1$: 近似硬标签分布
- $T=3$: 软化概率分布，让CNN学习类别间的相似性
- $T^2$因子: 补偿因温度缩放导致的KL散度量级变化

### 6.2 数据集详情

#### PAMAP2
- **来源**: 佩饰传感器数据集，8名用户
- **类别**: 下楼(0)、坐着(1)、站立(2)、走路(3)、慢跑(4)
- **划分**: Train 2,137 / Test 535
- **特点**: 包含日常活动和强度活动

#### UCI-HAR
- **来源**: 智能手机传感器，30名志愿者
- **类别**: 走路、上楼、下楼、坐着、站立、躺着
- **划分**: Train 7,352 / Test 2,947
- **特点**: 经典基准数据集，类别相对均衡

#### MotionSense
- **来源**: iPhone6s 传感器，24名用户
- **类别**: 下楼(0)、慢跑(1)、坐着(2)、站立(3)、上楼(4)、走路(5)
- **划分**: Train 17,492 / Test 4,373
- **特点**: 包含不同活动和实验环境

#### WISDM
- **来源**: Android手机，29名用户
- **类别**: 走路、慢跑、上楼、下楼、坐着、站立
- **划分**: Train 13,365 / Test 3,342
- **特点**: 用户级别划分（按用户划分train/test）

#### KuHar
- **来源**: 手腕和胸部传感器，90名受试者
- **类别**: 18类（Stand, Sit, Talk-sit, Talk-stand, Stand-sit, Lay, Lay-stand, Pick, Jump, Push-up, Sit-up, Walk, Walk-backwards, Walk-circle, Run, Stair-up, Stair-down, Table-tennis）
- **划分**: Train 20,000 / Test 19,129
- **特点**: **类别最多的数据集**，包含大量相似动作

#### UCI-HAR-New
- **来源**: UCI改进版，加入了过渡态动作
- **类别**: 12类（基础6类 + STAND_TO_SIT, SIT_TO_STAND, SIT_TO_LIE, LIE_TO_SIT, STAND_TO_LIE, LIE_TO_STAND）
- **划分**: Train 6,213 / Test 3,162
- **特点**: 包含过渡态动作，**识别难度最大**

#### MotionSense-DM
- **来源**: iPhone DeviceMotion API
- **类别**: downstairs, jogging, sitting, standing, upstairs, walking
- **划分**: Train 13,777 / Test 4,306
- **特点**: 12通道（加速度9+陀螺仪9，筛选后12维）

#### HARTH
- **来源**: 下背部IMU + 大腿IMU
- **类别**: 左立(5)、走路(1→3)、上楼(4)、下楼(2)、右立(0)、站立(6)
- **划分**: Train 20,000 / Test 17,678
- **特点**: 双传感器配置，6类日常活动

#### Gait
- **来源**: 脚踝加速度传感器
- **类别**: 慢速走(1)、正常走(2)、站立(3)、活动(4)
- **划分**: Train 588 / Test 184
- **特点**: **样本量最小**，步态分类

### 6.3 数据预处理

- **窗口化**: 128采样点窗口，步长64（50%重叠）
- **缺失值处理**: 丢弃含NaN的窗口
- **归一化**: Z-score标准化（按通道计算mean/std）
- **数据增强**: Mixup + Gaussian Noise（见4.3节）

## 10. 关键发现与分析

### 10.1 蒸馏效果与数据集复杂度的关系

```
准确率天花板（越难提升）:
WISDM(99.6%) ≈ MotionSense-DM(99.6%) > MotionSense(99.4%) > UCI-HAR(96.5%) > HARTH(95.8%) > PAMAP2(93.1%) > UCI-HAR-New(93.7%) > Gait(98.4%) > KuHar(81.0%)
                                                        ← 已饱和 ──→  ← 中间 ──→  ← 有空间 ──→  ← 最需优化 ──→
```

**发现**: 蒸馏效果与数据集难度呈**负相关**趋势：
- 高饱和数据集（WISDM, MotionSense-DM）→ 几乎无提升
- 中等难度（PAMAP2, UCI-HAR）→ 小幅提升
- 高难度+多类（KuHar 18类, UCI-HAR-New 12类过渡态）→ 提升不稳定（有时剧烈波动）

### 10.2 过渡态动作是共同难点

UCI-HAR-New的过渡态实验表明：
- SIT_TO_STAND和LIE_TO_STAND在蒸馏后完全失效
- 原因：过渡态时间窗口短，物理特征不明显，MiniMax软标签可能confident但错误
- **建议**: 后续应使用更长的上下文窗口或时序模型处理过渡态

### 10.3 Sit/Stand混淆是核心挑战

KuHar实验中Sit和Stand的相互干扰最严重：
- Sit: +37.56%
- Stand: -41.25%
- 两者物理特征相似（静止、低频），蒸馏软标签无法可靠区分

### 10.4 小数据集蒸馏价值更大

Gait实验（训练仅588样本）显示：
- +1.63%提升，蒸馏效果最显著
- 说明MiniMax的物理常识知识对稀缺数据帮助更大

### 10.5 软标签质量依赖Prompt工程

当前Prompt模板：
- 有效利用了acc_mag, y_mean, peaks, dom_freq等物理特征
- MiniMax的"Physics"提示有助于理解上下楼的Y轴差异
- 但对过渡态动作和相似姿态区分能力有限

## 11. 结论

### 11.1 主要结论

1. **MiniMax知识蒸馏在9个数据集上平均提升 +0.32%**，其中Gait提升最显著(+1.63%)，WISDM/MotionSense-DM已达饱和
2. **蒸馏对小数据集和复杂多类场景效果更明显**，如Gait(+1.63%)、KuHar Sit类(+37.56%)
3. **过渡态动作（UCI-HAR-New）是共同挑战**，SIT_TO_STAND/LIE_TO_STAND蒸馏后完全失效
4. **相似动作区分（Sit/Stand）是核心难点**，软标签无法可靠解决这类问题
5. **高饱和数据集无提升空间**，WISDM/MotionSense-DM接近100%后蒸馏几乎没有帮助

### 11.2 局限性

- 软标签生成依赖MiniMax API，成本和延迟不可忽视
- 过渡态动作的短窗口特征不足以被1D CNN捕获
- KuHar 18类场景的类别平衡和软标签质量需进一步优化
- 当前仅用传感器时序数据，未融合视觉或文本描述

### 11.3 未来工作方向

1. **时序建模**: 引入LSTM/Transformer处理过渡态动作
2. **多模态**: 融合视觉（摄像头）或文本（活动描述）信息
3. **主动学习**: 优先对难分类样本调用MiniMax API
4. **课程蒸馏**: 从易到难逐步蒸馏，先简单类后复杂类
5. **自蒸馏**: 用Pure CNN生成软标签进行自训练
