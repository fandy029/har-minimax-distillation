## 4.2 实验设计

### 4.2.1 对比维度

本文设计了三个维度的对比实验：

**维度一：模型架构对比。** 在纯硬标签训练下（无蒸馏），比较三种学生模型在各数据集上的基线性能：
- Pure CNN（无蒸馏基线）
- CNN-Residual（无蒸馏基线）
- Transformer（无蒸馏基线）

**维度二：蒸馏参数搜索。** 对于每种模型架构，搜索多组温度 T 和权重 α 组合，找到各数据集上的最优配置。T 的取值范围覆盖 1.0 到 3.5，α 从 0.1 到 0.9，覆盖从"高度依赖软标签"到"接近纯硬标签训练"的整个区间。

### 4.2.2 数据集覆盖

5 个数据集的分类难度和特点差异明显，能覆盖不同的实验场景：

- Gait（4 类，737 训练窗口）：最小数据集，类别严重不平衡。检验各架构在小样本条件下的表现。
- PAMAP2（5 类，7,613 训练窗口）：中等规模，物理特征可分性好。检验蒸馏在"本来就不难"的数据集上是否还能提供额外增益。
- MotionSense（6 类，15,304 训练窗口）：使用去重力加速度，静态姿势区分有难度。检验不同架构对这种传感器预处理的适应性。
- HARTH（8 类，65,181 训练窗口）：双传感器、严重不平衡。是评估 Focal Loss + 蒸馏对不平衡问题处理效果的核心场景。
- KuHar（18 类，66,946 训练窗口）：类别最多、规模最大。检验 Transformer 的长距离依赖优势能否在大量类别场景中体现，同时也考察蒸馏框架在多类场景下的鲁棒性。

### 4.2.3 实验流程

对于每个数据集，完整的实验流程为：

1. 运行 prepare 脚本，完成数据加载、窗口分割、数据集划分
2. 运行 gen 脚本，按类并行调用 MiniMax API 生成软标签
3. 运行 merge 脚本，合并各类别的软标签为完整标签文件
4. 阶段 1 训练：分别用 Pure CNN、CNN-Residual、Transformer 做无蒸馏训练，记录基线准确率
5. 阶段 2 训练：对每种架构加载阶段 1 最佳权重，用不同的 (T, α) 参数组合做蒸馏训练，记录所有组合的结果
6. 汇总各数据集上各架构在不同参数下的最优结果进行对比分析

### 4.3.3 分析框架

结果分析将围绕以下几个角度展开：

（1）**架构对蒸馏增益的影响。** 比较三种架构在蒸馏前后的提升幅度，分析哪种架构从软标签中获益最多。初步来看，CNN-Residual 因为残差连接保留了更丰富的底层特征，可能对软标签中的细节信息吸收更好；Transformer 在大数据集上可能因全局注意力优势而表现突出。

（2）**数据集特性与参数选择的关系。** 不同数据集的最优 (T, α) 是否存在规律。例如，类别多、数据量大的 KuHar 是否偏向较高温度（T 较大，软标签更平坦），类别少、类别间分离度好的 Gait 是否偏向较低温度（T 较小，软标签更锐利）。

（3）**计算开销与实际部署可行性。** 三种架构的参数量、推理延迟与精度的权衡，为嵌入式部署提供参考。

## 4.4 本章小结

本章设计了系统的实验方案，从模型架构和蒸馏参数两个维度进行对比。5 个数据集覆盖了从小样本（Gait，737 窗口）到大规模（KuHar，66,946 窗口）、从少类（Gait，4 类）到多类（KuHar，18 类）的广泛场景。

三种学生模型架构在参数量和特征提取机制上各有侧重——Pure CNN 轻量高效，CNN-Residual 训练更稳定，Transformer 擅长全局时序建模。蒸馏框架通过温度 T 和权重 α 提供了对硬软标签监督比例的控制。
