## 参数选择分析

### 为什么软标签占比较高 (40-60%)？

| 数据集 | 软标签 | 原因 |
|--------|--------|------|
| Gait | 50% | LLM准确率79%，类别少，软标签提供的类别间关系信息价值高 |
| PAMAP2 | 40-50% | 基线极高，需correct_only排噪后用中等软标签占比微调 |
| MotionSense | 50-60% | userAcceleration去重力后静态姿势难分，LLM对静态类接近100%，软标签价值高 |
| HARTH | 50-60% | 极度不平衡(sit 29k vs stairs_down 682)，软标签帮助少数类学习类别间相似性 |
| KuHar | 50-60% | 18类特征严重重叠，需要更多软标签信息来区分过渡活动(Talk-sit等) |

### α 和 T 的选择规律

1. **类别多/重叠严重 → 低α + 低T**: KuHar需要更多软标签(α=0.4-0.5)和尖锐分布(T=2.5)来学习细粒度类间关系
2. **基线高/类间分离好 → 高α**: PAMAP2需要α=0.5-0.6防止软标签噪声
3. **T 与软标签质量正相关**: LLM准确率越高的数据集,可以用越高的T来平滑软标签,提取结构性知识
4. **Transformer 倾向于 all 策略**: 自注意力已捕获类间关系,不需要 filtered/correct_only 的额外筛选

### 蒸馏收益与数据集特征

| 因素 | 收益 | 代表 |
|------|------|------|
| 类别多、特征重叠 | **高 (+4-5pp)** | KuHar |
| 类别不平衡严重 | **中高 (+3pp)** | HARTH |
| 传感器预处理(去重力) | **中 (+2pp)** | MotionSense |
| 类别少、分离好 | **低 (+0.5-2pp)** | Gait, PAMAP2 |

> 蒸馏增益与任务的固有难度正相关——软标签在模型"需要帮助"的地方提供最大价值。
