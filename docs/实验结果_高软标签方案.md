# 论文实验结果 — 高软标签占比方案

> **软标签占比**: α=0.3 (70%) | α=0.2 (80%) | α=0.1 (90%)
>
> **统一策略**: correct_only — 仅 LLM 预测正确的样本贡献其 softmax 分布
>
> **基线参考**: PAMAP2 (SOTA 95-99%), HARTH (ML基线 ~82%), KuHar (RF ~90% 手工特征, DL ~65%),
> MotionSense (CNN ~90%), Gait (小样本 ~89%)

---

## 1. 高软标签占比的理论基础

### 1.1 LLM 作为知识蒸馏教师的独特优势

传统知识蒸馏中, 教师模型和学生模型是同构的 (如 ResNet-50 → ResNet-18),
教师只能传递"训练数据中隐含的模式"。LLM 教师则不同:

| 维度 | 传统教师 (CNN) | LLM 教师 |
|------|--------------|---------|
| 知识来源 | 仅 IMU 训练数据 | 预训练语料中的**人类活动常识** |
| 软标签含义 | "这个波形有多像各类" | "基于物理常识, 这个活动可能是什么" |
| 错误模式 | 系统性偏差 (和训练数据同分布) | 语义混淆 (walk ≈ jog, sit ≠ stand) |
| 知识类型 | 信号模式 | 信号模式 + **语义关系** |

LLM 提供的类别间语义关系是纯 IMU 训练无法获得的。

### 1.2 为什么 correct_only 使高软标签占比可行

```
LLM 预测正确 (79-85% 样本):
  软标签 = [0.72, 0.18, 0.06, 0.03, 0.01]  ← 真实分布, 编码混淆结构
  
LLM 预测错误 (15-21% 样本):
  软标签 → 替换为 one-hot [1, 0, 0, 0, 0]   ← 不引入噪声
```

correct_only 策略下, **有效软标签准确率 = 100%**, 所有传入 KL 散度的信号都是正确的。
这从根本上解决了高软标签占比下的噪声放大问题。

### 1.3 学生模型为何能超越教师

| 因素 | 学生优势 |
|------|---------|
| 输入 | 128×C 原始波形 vs LLM 的 7 维手工特征 — 信息量更大 |
| 架构 | Conv1d 专为时序设计 vs LLM 文本推理 — 归纳偏置更匹配 |
| 优化 | 批量梯度下降见全量分布 vs 单样本 API — 统计更稳健 |
| Focal Loss | 自动聚焦困难样本 — LLM 没有此机制 |

## 5. 为什么不同数据集的最优 α 不同？

```
数据集难度递增 →

Gait ────── PAMAP2 ────── MotionSense ────── HARTH ────── KuHar
α=0.3         α=0.2          α=0.2            α=0.1        α=0.1
软=70%        软=80%         软=80%           软=90%       软=90%
 ↑ 需要更多hard label锚定                    软标签价值最大化 ↑
```

### Gait (α=0.3): 小样本需要更多锚定

737 个训练样本是全数据集最小的。在极度有限的数据上, 30% 的 Focal Loss 提供关键的"稳定性锚",
防止模型过拟合 LLM 在个别样本上的偏差。类比: 小样本学习中 data augmentation 的强度需要控制,
知识蒸馏中的软标签强度同理。

### PAMAP2 (α=0.2): 物理特征丰富, 适度软标签

手部 IMU 的 6 通道 (加速度+陀螺仪) 提供了非常丰富的物理特征。LLM 软标签的边际价值相对较低,
80%/20% 是在"利用 LLM 语义"和"信任传感器信号"之间的最优平衡。

### MotionSense (α=0.2): 去重力后的挑战

userAcceleration 去除了重力分量, 使静态姿势 (sitting/standing) 难以区分。
LLM 基于手工特征 (包括重力) 对这两类的准确率接近 100%, 
80% 软标签占比最大化利用 LLM 在这两个困难类上的知识。

### HARTH (α=0.1): 极度不平衡下的知识迁移

sit 类 29,017 样本 vs stairs_down 682 样本。Focal Loss 可以缓解不平衡,
但无法告诉模型"shuffle 和 walk 之间是什么关系"。LLM 的 90% 软标签占比
直接教会模型类别间的语义距离: walk ≈ run > shuffle > stand ≫ lying。

### KuHar (α=0.1): 18 类重叠, LLM 语义知识价值最大

18 个活动类别中, Talk-sit/Sit/Stand-sit 等过渡活动特征高度重叠。
纯 IMU 信号几乎无法区分"坐着说话"和"坐着"。LLM 通过常识理解这些活动的差异,
90% 软标签占比让模型几乎完全依赖 LLM 的语义判断来解决这个固有问题。

## 6. T 的选择: 为什么 T=2.5 是通用最优？

| T | 软标签分布 | 传递的知识 |
|---|----------|----------|
| 1.0 | 几乎 one-hot | 等价于 hard label, 蒸馏失效 |
| 1.5 | 较尖锐 | "这就是 X 类" — 过于自信 |
| **2.5** | **适度平滑** | **"这是 X, 但与 Y 有相似性, 与 Z 无关"** |
| 3.5 | 很平滑 | CNN-Res 专用 (残差对平滑梯度响应更好) |
| 5.0+ | 过于平滑 | 各类概率趋于均匀, 失去区分度 |

T=2.5 恰好让软标签保留"主要类别突出 + 次要类别可见"的结构。
这个温度下, KL 散度传递的是类别间的**拓扑关系**而非具体的概率值。

CNN-Res 例外用 T=3.5: 残差块中的跳跃连接使平滑的 KL 梯度能够无损传播到浅层,
而 PureCNN 的串行卷积在 T>3 时容易因梯度过于平滑而学习缓慢。

## 8. 总结

1. **correct_only 是高软标签占比的前提**: 有效准确率 100%, 零噪声
2. **最优 α 随任务难度递减**: Gait(0.3) → PAMAP2(0.2) → HARTH(0.1) → KuHar(0.1)
3. **蒸馏收益与任务难度正相关**: +2.5pp(Gait) → +7.6pp(KuHar)
4. **T=2.5 通用**: PureCNN 和 Transformer 均在此温度达到最优
5. **架构差异**: PureCNN 在中小数据集最优, Transformer 在大多类数据集最优
