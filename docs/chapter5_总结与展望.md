# 第5章 总结与展望

## 5.1 研究工作总结

从头到尾串一遍这条线：IMU 原始信号拿来，滑窗切成 128 步一段，算物理特征，拼 prompt 喂给 MiniMax 大模型，让它零样本输出概率分布当软标签；学生模型这边备了纯 CNN、CNN-Residual 和 Transformer 三套架构，先 Focal Loss 预训练一轮打基础，再加蒸馏微调吸收大模型的知识；最后在 Gait、PAMAP2、MotionSense、HARTH 和 KuHar 五个数据集上全跑了一遍。

软标签生成这块，五个数据集传感器配置不一样，类别数从 4 到 18 都有，但 prompt 设计思路是统一的。四条原则：prompt 里塞实际数据的 p50 参考值而不是模糊的文字描述，让大模型做数值比较；先粗分后细分，比如 KuHar 的 18 类先按能量拆成四个层级，每层 4-7 类，比较空间直接压到四分之一；特征选取跟物理规律挂钩，HARTH 靠大腿重力分量判断姿态，MotionSense 靠水平加速度方差区分坐和站，不是拍脑袋选的；最后逼着模型输出概率分布而不是 one-hot，类别间的相似关系才能保留下来。工程上搞了按类并行生成，每个类别独立进程跑，文件锁互斥，断点续传，不然五个数据集十几万个窗口串行跑不知道要等到什么时候。最终大模型的软标签准确率落在 78% 到 91% 之间，够用了。

训练策略上分了两步。第一步只用硬标签跑 Focal Loss 预训练，让模型先学会大致看信号。第二步载入预训练权重，学习率降下来，引入蒸馏损失。损失函数是两项加权——Focal Loss 继续拿硬标签压着别跑偏，KL 散度让学生输出分布去跟大模型给的软标签分布对齐。关键处理是软标签进训练前先过 correct_only 过滤：大模型判断对的保留原始分布，判断错的退化为硬标签。这样一来参与蒸馏的信号准确率百分之百，软标签占比推到 90% 也不怕灌噪声进去。

模型对比上三套架构共享同一套训练和蒸馏参数，比较算公平的。Pure CNN 四层 1D 卷积，核从 7 逐层降到 3，后面接自适应池化和全连接头，简单管用。CNN-Residual 加了两组残差块，理论上梯度可以跨层直传，但在四层深度下优势没发挥出来，参数多了反而拖了点后腿。Transformer 三层 Encoder-Only，自注意力一次扫完所有时间步，大类场景下比 CNN 强。最后的结果：Pure CNN 在中小数据集上表现最好（Gait 92.41%、PAMAP2 94.88%、MotionSense 93.15%），Transformer 在 HARTH（91.82%）和 KuHar（91.15%）上领先。

实验做了 3 种架构 × 3 组 α × 5 个数据集 = 45 组，蒸馏后全部有提升，没有退化的。增益从 PAMAP2 的 +1 个百分点到 KuHar 的 +7.23 个百分点，差了六个多点。拆收益来源的话：类别越多、特征重叠越严重，软标签里"类别间关系"的信息越值钱——KuHar 里走路、后退、转圈三种变体的波形几乎分不开，大模型靠常识知道它们不一样；类别不平衡越厉害，软标签对少数类的价值越大——HARTH 里 stairs_down 从 52% 提到了 67.5%；预处理丢了信息的时候，大模型能用手工特征补回来——MotionSense 去重力后坐和站在加速度上看几乎没区别，大模型靠重力方向判断，准确率接近 100%。

## 5.2 主要结论

大模型蒸馏对轻量 HAR 模型就是管用，而且是普适的。45 组实验每组都在涨，五个传感器配置、类别复杂度、数据规模各不相同的公开数据集上结果一致。说明"拿大模型的物理常识喂小传感器模型"这条路站得住。

而且任务越难蒸馏越有用。这不是玩文字游戏——直觉上可能觉得容易的任务好提，实际正好反过来。PAMAP2 不蒸馏已经 93.85%，蒸馏才加了 1 个点。KuHar 不蒸馏才 81.40%，蒸馏拉到 88.20%。大模型那点知识在模型自己搞不定的地方才值大钱——多类混在一起、少数类样本不够、预处理把关键信息丢了，这些缺口正好是软标签能填的。

α 选多少看数据集说话。Gait 就 737 个样本，最优 α=0.3，硬标签得留三成兜底。HARTH 五万九样本，最优 α=0.1，软标签占九成也没问题。α 本质上就是硬标签安全性和软标签信息量之间的天平——样本少的时候多留点硬标签防过拟合，样本多了放手信任软标签就行。

温度 T=2.5 在绝大多数场景下最好。除了 CNN-Residual 偶尔偏好 3.5，其他组合最优 T 全是 2.5。这个温度刚好让软标签处于"主类突出、次优类看得见"的状态——T=1 的时候软标签跟 one-hot 没区别，蒸馏名存实亡；T=5 以上分布拉平了，区分度也没了。

架构选什么跟数据规模走就行。Gait、PAMAP2、MotionSense 这种中小规模，Pure CNN 最稳。HARTH、KuHar 这种数据量大、类别也多的，Transformer 的全局注意力占了便宜。CNN-Residual 四层深度下残差的优势显不出来，参数反而成了负担。

## 5.3 局限与不足

成效归成效，该认的问题还是得认。

软标签生成太贵了。KuHar 五万七千个窗口，一个窗口调一次 API，0.3 秒间隔串行就要四个多小时。并行架构和断点续传缓解了一些，但跟传统教师网络一次前向就拿全量软标签比，成本差距还是摆在那里。要在新数据集上反复迭代蒸馏，时间和预算都卡脖子。

correct_only 过滤有点粗糙。大模型判断错的样本全部退化为硬标签，信号是干净了，但很多有价值的分布信息也一起扔了。比如一个走路的样本被大模型错判成跑步，但它给的分布是 Walk 0.35、Run 0.45、Jog 0.15——这其实已经传递了"走路和跑步在信号上确实像"这个信息。一刀切退化成 hard label 就把这部分丢了。能不能按置信度设一个软阈值而不是非黑即白的对错判断，值得再想想。

prompt 还是靠人工调。五个数据集各有各的模板，虽然统一了原则，但分层深度、特征选哪些、阈值设多少，全得对着数据分布手动试出来的。数据集一多这个投入就线性往上走。理想情况是大模型能自己根据反馈优化 prompt，但离那一步还有距离。

只用了 MiniMax M2.7 一个教师。没试过 GPT-4、Claude 或者其他模型来做的效果差多少。不同的模型预训练数据不同、对物理常识的掌握程度也不同，它们在 HAR 蒸馏上的表现很可能差距不小。而且单一教师有系统性偏差的问题——如果它对某几类的判断有偏好，学生模型也会跟着学歪。

学生模型偏浅。四层 CNN、三层 Transformer，设计意图是冲着边缘设备去的，但也意味着表达能力上限摆在那里。如果能用更深的结构（比如八层残差、混合专家头）在相近的计算预算内拿到更高的基线，蒸馏出来的结果应该也会更好。这块没展开做。

没做跨数据集泛化。实验全是数据集内部的随机划分——窗口独立同分布，训练和测试来自同一个样本池。但实际落地是你用一个位置训练好的模型去猜另一个位置的数据，传感器配置、佩戴位置、采样率全变了。现在这版没验证过能不能搬过去用。

窗口之间的时序关系浪费了。相邻窗口 50% 重叠率，显然在描述同一段运动的连续片段。但当前方案把所有窗口当独立样本处理，大模型推理也不看上下文，软标签完全依赖单窗口的静态特征。连续几个窗口的大模型判断结果其实是前后一致的，这个信息没用上。

困难类的蒸馏效果不对称。大模型对 Lay、Run、Jump 这些特征鲜明的类判断准确率很高，对 Walk-backwards、Talk-sit 这些模糊类准确率明显偏低。correct_only 一过，最需要蒸馏帮助的困难类反而从软标签获益最少——因为它的大部分样本被退化为硬标签了。这是个内在矛盾。

## 5.4 未来工作展望

局限摆出来了，往哪走也比较清楚。

多教师融合可以试试。不同大模型在预训练数据上的差异可能形成互补——一个对过渡活动判断准，另一个对稳态活动分得好。把它们的软标签做个加权融合，或者按类别选最优教师，应该比单打独斗强。实现上不复杂：同样一组窗口分别跑几个模型，按置信度或准确率加权平均就行。

让大模型自己写 prompt。人工调 prompt 的边际成本太高了。一个自然的想法是让大模型自己根据反馈来迭代——给个通用模板和几个示例窗口，产出的软标签根据跟真实标签的匹配度自动调整特征选择和层级结构。再往前走一步，能不能直接喂原始波形而不是手工特征，让大模型自己从数值里看规律？这要求大模型有很强的数值理解能力，一时半会不一定行，但方向是对的。

窗口时序关系不该浪费。一个直接的想法是用滑动窗口做一致性投票：前三个窗口大模型判为走路，后三个也是走路，中间这个窗口就算原始输出有点飘，也能靠上下文拉回来。或者把时间连贯性编码进蒸馏损失——鼓励模型对相邻窗口的输出保持平滑，软标签变化太大就罚一点。这事实现成本不高，应该能见到效果。

按质量分配蒸馏权重。目前的 correct_only 是二元判断——留或扔。可以改成连续权重：置信度高的给大权重，低的给小权重甚至退化为硬标签。再往前走，对大模型判断错的样本，可以把分布中的峰值纠正为真实标签，但非峰值的分布信息保留——既改了方向又没扔信息。

上硬件跑一下。学生模型设计再轻量，终归是在服务器上验证的。下一步该推到真正的小设备上——Coral Edge TPU、STM32、手机 NPU——测一下真实延迟、内存和功耗。再结合 INT8 量化、剪枝和网络结构搜索，在蒸馏保住的精度基础上继续往下压。

跟半监督学习结合。现在的软标签生成得有标注数据才能算 p50。但真实场景里大量传感器数据根本没标签——用户全天戴着设备，大部分时间你不知道他在干什么。如果能用蒸馏策略给未标注数据生成伪标签，再套上一致性正则化（FixMatch 或者 Mean Teacher），就能把没标签的数据也用起来，进一步压减对标注的需求。

试跨数据集迁移。在一个数据充足的数据集上（比如 KuHar）蒸馏出模型，直接扔到另一个传感器配置和佩戴位置不同的数据集上微调。如果蒸馏学到的是"运动的一般规律"——什么信号模式对应什么类型的活动——那它就应该能跨数据集泛化。能成的话，蒸馏就可以成为 HAR 领域的一种通用知识库构建手段。

搞在线蒸馏。现在是离线生成软标签再训练，但实际系统里用户会不断产生新数据。对新增数据快速调 API、增量更新模型，这个流程得跑通。可以做缓存——新窗口的特征跟历史窗口接近就直接复用历史软标签，只有特征偏移了才触发新一轮 API 调用。

## 5.5 本章小结

绕了一圈回到起点：用大语言模型的知识给轻量 IMU 模型赋能，这条路走得通。五个数据集、45 组实验、全部正增益，没有退化的。特别是类别多、样本少、信号模糊的时候，大模型的"常识"正好能补上纯数据驱动够不到的缺口。prompt 设计的四个原则、两阶段蒸馏框架、correct_only 策略、α 和 T 的调参经验，后续做传感器时序分析的跨模态知识迁移应该能直接用上。

当然也还有不小的距离要跑。软标签生成太贵、prompt 靠人工、跨数据集验证还没做——这些坎都得迈过去。不过从数据来看，方向没问题——大语言模型能做的远不止聊天和写代码，它理解物理世界的那套东西，放在传感器信号分析上也有正经用途。
