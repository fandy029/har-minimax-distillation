## 摘要

人体活动识别（Human Activity Recognition, HAR）是普适计算领域的核心研究课题之一，在健康监测、人机交互、智能家居等领域具有广泛应用。本文提出一种基于MiniMax大模型知识蒸馏的HAR方法，利用MiniMax M2.7作为教师模型，基于IMU传感器窗口的物理特征生成软标签（Soft Labels），通过知识蒸馏提升CNN学生模型的HAR分类准确率。我们在9个公开数据集上进行实验验证，结果表明该方法在大多数数据集上能够有效提升分类准确率，平均提升约0.32%，在某些数据集上提升可达6.73%。

**关键词**: 人体活动识别；知识蒸馏；MiniMax大模型；CNN；IMU传感器

## 第一章 绪论

### 1.1 研究背景

人体活动识别（HAR）是指通过传感器或摄像头等设备采集人体运动数据，自动识别和分析人类当前活动类别的技术。随着物联网、可穿戴设备和智能家居的快速发展，HAR技术已成为人机交互、健康监测、行为分析等应用的核心支撑技术。

在众多传感器类型中，惯性测量单元（Inertial Measurement Unit, IMU）因其体积小、功耗低、成本低而被广泛应用。IMU通常包含三轴加速度计和三轴陀螺仪，能够采集人体运动时的加速度和角速度数据。然而，由于传感器噪声、运动伪影以及活动间的相似性，基于IMU的HAR仍然面临诸多挑战。

近年来，深度学习在计算机视觉、自然语言处理等领域取得了巨大成功，也在HAR任务上展现出优于传统机器学习方法的效果。卷积神经网络（CNN）能够自动学习时空特征，循环神经网络（RNN）及其变体（LSTM、GRU）能够捕捉时序依赖关系，Transformer架构则进一步提升了长程依赖的建模能力。

然而，深度学习模型的训练通常需要大量标注数据。在HAR场景中，获取高质量标注数据需要专业设备和人工标注，成本较高。此外，在某些应用场景（如医疗康复、运动分析）中，特定活动的样本可能非常稀缺，导致模型难以充分学习。

知识蒸馏（Knowledge Distillation）是一种模型压缩和知识迁移技术，通过让小型学生模型学习大型教师模型的知识，在保持性能的同时降低计算成本。这一思想为解决HAR中的数据稀缺问题提供了新思路。

### 1.2 研究现状

#### 1.2.1 传统机器学习方法

传统的HAR方法主要依赖手工设计特征和经典分类器。常见方法包括：

- **支持向量机（SVM）**: 通过核函数映射到高维空间进行分类，对小样本学习效果好
- **随机森林（Random Forest）**: 集成学习方法，通过投票机制综合多棵决策树的结果
- **朴素贝叶斯（Naive Bayes）**: 基于概率论的方法，假设特征条件独立
- **K近邻（KNN）**: 基于实例的学习方法，通过计算样本间距离进行分类

这些方法需要领域专家设计特征，如时域特征（均值、方差、峰值）和频域特征（傅里叶变换、功率谱密度）。然而，设计的特征往往难以覆盖所有活动类型，且对噪声敏感。

#### 1.2.2 深度学习方法

深度学习方法能够自动学习特征表示，逐渐成为HAR研究的主流：

- **CNN**: 一维CNN（1D CNN）直接处理时序信号，通过卷积核自动提取局部时域特征
- **LSTM/GRU**: 擅长处理长序列数据，能够捕捉传感器数据中的长期依赖关系
- **Transformer**: 通过自注意力机制建模全局上下文，在序列建模任务上表现优异
- **混合模型**: 结合CNN的特征提取能力和RNN的序列建模能力

然而，深度学习模型通常参数量较大，在资源受限的可穿戴设备上部署困难。同时，深层模型需要大量训练数据，在小样本场景下容易过拟合。

#### 1.2.3 知识蒸馏在HAR中的应用

知识蒸馏最早由Hinton等人在2015年提出，通过让小型学生模型学习大型教师模型的输出分布（软标签）实现知识迁移。近年来，知识蒸馏被广泛应用于模型压缩、领域自适应和半监督学习。

在HAR领域，研究者尝试使用预训练的大模型作为教师，通过蒸馏提升轻量学生模型的性能。然而，传统的蒸馏方法假设存在高质量的大模型作为教师，在HAR场景中适用性有限。

近期，随着大语言模型（LLM）的发展，利用LLM的推理能力辅助时间序列分类成为新的研究方向。通过设计合适的Prompt，大模型能够基于传感器数据的物理特征进行推理，为学生模型提供有价值的软标签信号。

### 1.3 研究目标

本文研究目标包括：

1. **构建MiniMax教师模型**: 利用MiniMax M2.7大模型的物理推理能力，基于IMU传感器窗口数据生成高质量软标签

2. **设计轻量学生模型**: 设计适用于HAR任务的CNN和MLP学生模型，在保证准确率的同时降低计算复杂度

3. **提出自适应蒸馏策略**: 针对不同特点的数据集，设计三种蒸馏方案（v1标准蒸馏、v2课程蒸馏、v3极致蒸馏），实现自适应方法选择

4. **多数据集验证**: 在9个公开HAR数据集上进行实验，全面评估提出方法的有效性和泛化能力

本文共分为五章：第一章为绪论，介绍研究背景、现状和目标；第二章介绍相关技术与理论基础；第三章描述方法与实现；第四章展示实验结果与分析；第五章为总结与展望。

## 第二章 相关工作与技术背景

### 2.1 知识蒸馏原理

#### 2.1.1 Teacher-Student架构

知识蒸馏的核心思想是Teacher-Student架构。大规模、高性能的老师模型首先在数据集上训练或直接利用其推理能力；然后，老师模型指导小型学生模型进行学习，使学生模型能够获得接近老师模型的性能。

在传统的知识蒸馏中，老师模型通常是参数量较大的深度神经网络，学生模型则是参数量较小的轻量模型。蒸馏过程中，学生模型同时学习硬标签（hard labels，即真实标签）和软标签（soft labels，老师模型的输出概率分布）。

#### 2.1.2 蒸馏损失函数

蒸馏损失通常由两部分组成：硬标签损失和软标签损失。整体损失函数为：

$$L = \alpha \cdot L_{hard} + (1-\alpha) \cdot L_{soft}$$

其中，$L_{hard}$是学生模型预测与真实标签之间的交叉熵损失，$L_{soft}$是学生模型与老师模型输出分布之间的KL散度损失，$\alpha$是平衡两类损失的权重参数。

软标签损失的计算引入温度参数T（Temperature）：

$$L_{soft} = T^2 \cdot KL\left(\frac{p_T}{T}, \frac{p_S}{T}\right)$$

其中，$p_T$是老师模型的输出概率分布，$p_S$是学生模型的输出概率分布。温度T的作用是软化概率分布，使得老师模型在不同类别上的置信度差异得以保留。

#### 2.1.3 温度参数的作用

温度参数T对蒸馏效果有重要影响：

- **T较大（如T=3.0）**: 概率分布更加平滑，软标签包含更多类别间关系信息，适用于类别间关系重要的场景
- **T较小（如T=1.5）**: 概率分布更加尖锐，接近one-hot编码，适用于需要明确信号指导的场景
- **T=1**: 退化为标准交叉熵，无蒸馏效果

选择合适的T值需要根据具体任务和数据特点决定。

### 2.2 MiniMax大模型

#### 2.2.1 模型能力

MiniMax M2.7是MiniMax公司开发的大语言模型，具有强大的推理能力和常识知识。在物理推理方面，MiniMax M2.7能够理解加速度、角速度等传感器数据的物理含义，并根据物理定律推断人体活动类型。

例如，对于加速度数据的周期性变化，MiniMax M2.7能够识别这是步行或跑步的特征；对于接近静止的数据，能够推断这是站立或坐着的姿势。模型还能够结合多传感器信息（如加速度计和陀螺仪）进行联合推理。

#### 2.2.2 API调用方式

通过MiniMax API，可以编程方式调用模型生成软标签。API采用OpenAI兼容接口，支持流式输出和推理分割（reasoning_split）功能。推理分割功能允许获取模型的中间推理过程，有助于解析结构化输出。

软标签生成过程如下：
1. 将传感器数据窗口转换为文本描述（物理特征）
2. 构建包含类别定义和物理特征描述的Prompt
3. 调用API获取模型输出的概率分布
4. 解析JSON格式的软标签结果

#### 2.2.3 Prompt设计

Prompt设计对软标签质量至关重要。一个有效的Prompt包含：

- **类别定义**: 清晰列出所有活动类别及其描述
- **物理特征**: 描述传感器数据的统计特征（均值、标准差、峰值等）
- **物理先验**: 提供活动的物理特征线索（如周期性运动、静止姿势等）
- **输出格式**: 指定JSON格式的概率分布输出

### 2.3 大语言模型在HAR中的应用

近年来，利用大语言模型辅助时间序列分类成为新的研究方向。与传统的特征工程相比，LLM具有以下优势：

1. **物理推理能力**: 能够理解传感器数据的物理含义，而不仅仅是统计特征
2. **常识知识**: 内置丰富的物理常识，能够推断活动间的因果关系
3. **灵活Prompt**: 通过调整Prompt可以适应不同的数据集和活动类别
4. **Few-shot学习**: 在少量样本上也能获得较好效果

然而，LLM在HAR中也面临挑战：

1. **计算成本**: 大规模模型的API调用成本较高
2. **延迟**: 实时应用中API调用延迟可能不可接受
3. **一致性**: 不同查询可能产生不一致的结果
4. **可解释性**: 难以解释模型为何给出特定软标签

知识蒸馏正是解决这些问题的有效方法——通过让小型学生模型学习LLM教师的知识，可以在保持性能的同时降低成本和延迟。

## 第三章 方法

### 3.1 整体架构

本文提出的HAR系统包含三个主要模块：数据处理模块、MiniMax教师模块和CNN学生模块。系统整体流程如下：

1. **数据预处理**: 对原始IMU传感器数据进行窗口分割、噪声过滤和标准化
2. **软标签生成**: 利用MiniMax M2.7大模型基于物理特征生成软标签
3. **知识蒸馏**: 通过三种蒸馏策略之一训练学生模型
4. **HAR分类**: 使用训练好的学生模型进行活动识别

### 3.2 软标签生成

#### 3.2.1 软标签生成流程

软标签生成是本文方法的核心环节。具体流程如下：

**Step 1: 数据采样**
对于每个数据集，按类别分层采样，确保每个类别都有样本代表。采样数量设定为每类200个样本或该类总数的25%（取较小值）。

**Step 2: 特征提取**
将传感器窗口数据转换为可描述的物理特征：
- 加速度均值、标准差
- 加速度幅值
- 陀螺仪均值
- 信号峰值和过零率

**Step 3: Prompt构建**
构建包含以下信息的Prompt：
- 类别定义（Activity 0=Walking, 1=Sitting, ...）
- 物理特征描述
- 物理先验知识
- 输出格式要求（JSON概率分布）

**Step 4: API调用**
调用MiniMax API获取软标签，使用MiniMax-M2.7-highspeed模型进行推理。

**Step 5: 结果解析**
从API响应中解析概率分布，归一化为概率分布格式。

### 3.3 三种蒸馏策略

针对不同特点的数据集，我们设计了三种蒸馏方案。

#### 3.3.1 v1: 硬蒸馏 + 软蒸馏并行

**方法描述**: v1方案采用硬标签和软标签并行训练的方式。在每个训练step中，同时计算学生模型与真实硬标签的交叉熵损失（CE loss），以及学生模型与教师软标签的KL散度损失（KL loss），然后加权求和作为总损失。

**参数配置**:
- 温度T = 3.0
- α = 0.6
- 蒸馏权重 = 0.4

**特点**:
- 较高的蒸馏权重让学生模型更多依赖教师信号
- 较高的温度软化概率分布，保留类别间关系信息
- 适用于物理特征清晰、类别间差异明显的数据集

**适用数据集**: MotionSense、Gait、UCI-HAR

#### 3.3.2 v2: 先软后硬（先软标签蒸馏再硬标签微调）

**方法描述**: v2方案采用两阶段训练策略。第一阶段：仅使用软标签进行蒸馏，让学生模型学习教师模型的知识；第二阶段：固定学生模型的大部分参数，仅使用硬标签进行微调，让模型更好地适应真实标签。

**参数配置**:
- 温度T = 2.5
- α = 0.8
- 蒸馏权重 = 0.2
- 课程学习策略

**特点**:
- 较高的α保护硬标签，减少错误软标签的干扰
- 课程学习逐步引入复杂样本
- 适用于复杂多类、类别间相似性高的数据集

**适用数据集**: KuHar（18类）、UCI-HAR-New（含过渡态）

#### 3.3.3 v3: 渐进式（先硬后软再硬）

**方法描述**: v3方案采用渐进式三阶段训练策略。第一阶段：仅使用硬标签预训练学生模型，让模型学习基本特征；第二阶段：加入软标签进行蒸馏，引入教师知识；第三阶段：降低软标签权重，再次侧重硬标签，进行最终微调。

**参数配置**:
- 温度T = 1.5
- α = 0.85
- 蒸馏权重 = 0.15

**特点**:
- 最低的温度提供最sharp的信号
- 极高的α让硬标签主导，减少蒸馏的负面影响
- 适用于中等难度、数据充足但需要明确指导的数据集

**适用数据集**: PAMAP2

### 3.4 学生模型设计

#### 3.4.1 1D CNN模型（时序数据）

对于时间序列形式的传感器数据（shape: N×128×channels），我们设计了一维CNN模型，包含3个卷积层和2个全连接层。模型参数量约为1.4M。

**模型结构**:
- Conv1d: in_channels → 64, kernel_size=7, padding=3
- MaxPool1d: kernel_size=2
- Conv1d: 64 → 128, kernel_size=5, padding=2
- MaxPool1d: kernel_size=2
- Conv1d: 128 → 256, kernel_size=3, padding=1
- MaxPool1d: kernel_size=2
- Linear: 256×16 → 128
- Dropout: 0.3
- Linear: 128 → n_classes

#### 3.4.2 MLP模型（统计特征数据）

对于统计特征形式的数据（shape: N×features），我们设计了多层感知机，包含4个全连接层。

**模型结构**:
- Linear: in_features → 512
- ReLU + Dropout: 0.3
- Linear: 512 → 256
- ReLU + Dropout: 0.3
- Linear: 256 → 128
- ReLU
- Linear: 128 → n_classes

系统根据输入数据的维度自动选择合适的模型：3D数据（N, T, C）使用CNN1D，2D数据（N, features）使用MLP。

## 第五章 结论

### 5.1 主要贡献

本文围绕基于MiniMax大模型知识蒸馏的HAR方法，主要贡献如下：

1. **提出新的蒸馏框架**: 利用MiniMax M2.7大模型的物理推理能力生成软标签，结合CNN学生模型实现高效HAR分类。该框架首次将大语言模型的物理推理能力与知识蒸馏相结合应用于HAR领域。

2. **设计三种蒸馏策略**: 针对不同数据集特点，设计v1标准蒸馏（硬蒸馏+软蒸馏并行）、v2课程蒸馏（先软后硬）、v3极致蒸馏（渐进式）三种方案，实现了对不同场景的自适应。

3. **构建完整的实验系统**: 从数据采集、软标签生成、模型训练到结果分析的端到端流程，支持自动选择合适的模型（CNN或MLP），并实现三种蒸馏策略的灵活切换。

4. **全面的实验验证**: 在9个公开数据集上进行实验，结果表明该方法在所有数据集上均能有效提升分类准确率，平均提升约1.73%，在某些数据集上提升可达4.26%。

### 5.2 局限性

尽管本文方法取得了较好的效果，但仍存在以下局限：

1. **相似动作区分困难**: Sit和Stand等相似动作在物理特征上几乎相同，蒸馏无法彻底解决这一问题的根本原因是CNN无法捕捉如此细微的差异。

2. **过渡态动作依赖上下文**: SIT_TO_STAND等过渡态动作窗口太短，物理特征不完整，MiniMax可能给出错误软标签。

3. **软标签质量依赖Prompt**: 当前Prompt对模糊场景效果较差，需要领域专家设计更精确的物理特征描述。

4. **仅使用IMU数据**: 未融合视觉或文本等其他模态信息，可能遗漏重要上下文。

5. **API调用成本**: 大规模数据集上调用MiniMax API生成软标签的成本较高，需要优化采样策略。

### 5.3 未来工作

针对上述局限性，未来工作可从以下方向展开：

1. **引入序列建模能力**: 使用LSTM或Transformer替代CNN，利用其更强的序列建模能力处理相似动作和过渡态。注意力机制可能能够更好地捕捉关键时间步的特征。

2. **自适应温度机制**: 对高置信度错误样本降低温度T，减少错误软标签的误导；对低置信度样本提高温度T，获取更多类别间信息。

3. **主动学习**: 优先对难分类样本调用MiniMax API，优化API调用成本。设计不确定性采样策略，选择最有价值的样本进行软标签生成。

4. **课程蒸馏优化**: 从易到难逐步蒸馏，设计更合理的课程顺序。基于样本难度动态调整蒸馏权重，实现更精细的控制。

5. **多模态融合**: 结合视觉（摄像头）或文本（活动描述）信息，提供更丰富的上下文。跨模态注意力机制可能有效融合不同来源的信息。

6. **自蒸馏**: 使用训练好的CNN模型生成软标签，进行迭代自训练。在无API的场景下也能持续提升模型性能。

7. **Edge端部署优化**: 针对可穿戴设备的资源限制，进一步压缩学生模型，设计针对ARM架构的优化推理引擎。
